解决git push代码到github上一直提示输入用户名及密码的问题

发现问题

从github上克隆到本地，要push时，发现老是要输入账号密码，而且不能push。

分析问题

那是因为使用了克隆默认的http的方式，切换成ssh方式即可。

解决问题

先查看当前方式：

git remote -v

把http方式改为ssh方式。先移除旧的http的origin：

git remote rm origin

再添加新的ssh方式的origin：

git remote add origin git@github.com:yychuyu/linux-system-programming.git

改动完之后直接执行git push是无法推送代码的，需要设置一下上游要跟踪的分支，与此同时会自动执行一次git push命令:

git push --set-upstream origin master
